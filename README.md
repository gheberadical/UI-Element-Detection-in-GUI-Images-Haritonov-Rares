# UI Element Detection in GUI Images

Bachelor's Thesis — Haritonov Rares-Costin · Babeș-Bolyai University, 2026

Benchmarks 7 modern object detectors on 3 GUI screenshot datasets covering desktop, web, and mobile interfaces. All experiments are evaluated under the COCO metric suite (mAP @ IoU 0.50:0.95).

---

## Models compared

| Model | Family | Backbone |
|---|---|---|
| YOLOv8n | YOLO | CSPDarknet (CNN) |
| YOLO11n | YOLO | C3k2 (CNN) |
| YOLOv12n | YOLO | R-ELAN (CNN + attention) |
| YOLO26n | YOLO | Dual-branch (CNN) |
| Faster R-CNN R50-FPN v2 | Two-stage CNN | ResNet-50 FPN |
| RT-DETR-L | Transformer | HGNetV2 (CNN) |
| RF-DETR-M | Transformer | DINOv2 (ViT, self-supervised) |

## Datasets

| Dataset | UI Type | Classes | Description |
|---|---|---|---|
| [UICVD](https://github.com/MadaDicu/UICVD) | Desktop | 16 | Native desktop application screenshots collected for RPA research |
| [GenGUI](https://github.com/MadaDicu/GENGUI) | Web | 13 | Web UI images synthetically generated using ChatGPT |
| [VINS](https://github.com/sbunian/VINS) | Mobile | 13 | High-fidelity iPhone and Android app screenshots (CHI 2021) |

### Dataset setup

Clone or download each dataset repository and place them under `datasets/` as follows:

```
datasets/
  GenGUI/          ← contents of https://github.com/MadaDicu/GENGUI
  UICVD/           ← contents of https://github.com/MadaDicu/UICVD
  VINS Dataset/    ← contents of https://github.com/sbunian/VINS
```

Then run the preparation scripts to convert each dataset to COCO and YOLO formats and create train/val/test splits:

```bash
bash scripts/prepare_all.sh
```

Prepared data is written to `data/prepared/` (not committed to git).

## Quick start — Demo

**Requirements:** Python 3.10+, CUDA-capable GPU recommended (CPU works but is slow for RF-DETR).

```bash
# 1. Create and activate a virtual environment
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate

# 2. Install PyTorch (pick the right CUDA version from https://pytorch.org/get-started/locally/)
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121
# or cu124, cu128 — whichever matches your driver

# 3. Install all other dependencies
pip install -r requirements.txt

# 4. Install the uidet package
pip install -e .

# 5. Download model weights
python download_weights.py

# 6. Run the interactive demo
python demo/app.py
```

The demo opens in your browser automatically. Upload any GUI screenshot, select the type of interface (desktop / web / mobile) and a model, then click **Run detection**.

## Training and evaluation

> This section is only needed if you want to retrain a model from scratch or run evaluation on your own data. The demo works out of the box with the provided weights.

**Prerequisites:** complete the [Dataset setup](#dataset-setup) section first.

```bash
# Train a model
python -m uidet.train --config configs/experiments/yolov8n_uicvd_uicvd16.yaml

# Evaluate a trained model on the test set
python -m uidet.eval_coco --config configs/experiments/yolov8n_uicvd_uicvd16.yaml --split test

# Run inference on a single image
python -m uidet.predict \
    --weights results_v2/yolov8n_uicvd_uicvd16/weights/best.pt \
    --image path/to/screenshot.png \
    --out predictions.png
```

Each experiment config in `configs/experiments/` specifies the model, dataset, taxonomy, hyperparameters and output directory. Results are saved to `results_v2/<experiment_name>/`.

## Repository layout

```
configs/
  experiments/       One YAML per experiment (model + dataset + hyperparameters)

data/
  ontology/          Class taxonomy mappings between datasets
  prepared/          COCO + YOLO formatted datasets — generated by prepare_all.sh
                     (not committed — regenerate from datasets/)

datasets/            Raw downloaded dataset files — not committed, see Dataset setup

demo/
  app.py             Gradio web app for interactive inference

kaggle/              Kaggle/Colab training notebooks (RF-DETR, RT-DETR — require >4 GB VRAM)

results_v2/                      ← not committed — download via download_weights.py
  <model>_<dataset>_<taxonomy>/
    coco_eval_test/
      metrics.json   All COCO evaluation metrics
    weights/
      best.pt        Trained model checkpoint
  figures/           Generated charts and figures

scripts/
  prepare_all.sh         Runs all three dataset preparation scripts
  prepare_uicvd.py       Converts UICVD CSV annotations → COCO + YOLO format
  prepare_gengui.py      Converts GenGUI CSV annotations → COCO + YOLO format
  prepare_vins.py        Converts VINS Pascal VOC XML → COCO + YOLO format
  _prepare_common.py     Shared utilities for the preparation scripts
  summarize_runs.py      Scans results_v2/ and prints a summary table
  to_rfdetr_layout.py    Converts YOLO-format dataset to RF-DETR folder layout

src/uidet/
  train.py               Training entry point
  eval_coco.py           COCO evaluation script (writes metrics.json)
  predict.py             Single-image inference CLI
  models/
    base.py              Abstract Detector class and build_detector() factory
    yolo_ultra.py        Wrapper for all Ultralytics YOLO models + RT-DETR
    faster_rcnn_tv.py    Faster R-CNN wrapper (torchvision)
    rfdetr.py            RF-DETR wrapper (Roboflow)
    _coco_dataset.py     PyTorch Dataset for COCO-format data (Faster R-CNN training)
  data/
    adapters/            Dataset-specific readers (GenGUI CSV, UICVD CSV, VINS Pascal VOC)
    exporters/           Writers for COCO JSON and YOLO txt formats
    splits.py            Train/val/test split logic
    unified.py           Shared internal Annotation dataclass
  utils/
    io.py                YAML load/save helpers
    metrics.py           Metric utilities
    viz.py               Bounding box drawing (used by demo)

pyproject.toml           Package metadata and build config
requirements.txt         All Python dependencies
```
